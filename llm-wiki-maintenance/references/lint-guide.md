# Lint 健康检查指南

## 批量原则（禁止逐项循环）

1. 一次性执行全部 6 项 + 子项，收集所有发现
2. 完整报告一次性呈现（按严重程度排列）
3. 用户确认修复范围 → 批量修复
4. 修复后一次性验证全部 6 项
5. 确认零问题后单次 `git commit -m "[lint] 标题"`

## 6 项检查

| 检查项 | 子项（一次性全部执行） |
|--------|------------------------|
| **矛盾** | 术语一致性扫描 → 断言交叉对比 → 日期矛盾检测 |
| **过时声明** | 扫描全部页面 sources frontmatter → 对比 raw/ 文件修改时间 → 列出 wiki 断言可能过时的页面 |
| **孤立页** | 入链数 ≤ 2 的页面列表 |
| **断链 / 缺失页面** | 提取所有 `[[wikilink]]` → 验证目标存在 → 列出缺失 + 引用来源 |
| **缺失交叉引用** | 扫描 concepts/ 和 syntheses/ → 识别主题重叠但「相关」段无互链的页面对 → 建议添加 `[[wikilink]]` |
| **数据缺口** | 扫描「待补充」「疑问」标记（排除"还可以了解更多"型缺口） |

## 报告要求

按严重程度排列。末尾追加「建议方向」：值得深入的概念、需新资料补充的缺口、值得研究的新问题。

## 规则

- 不自动删除任何内容
- 修复前需用户确认
- 发现矛盾 → 按 Ingest 矛盾处理执行（保留双方 + contested:true + 告知用户）
