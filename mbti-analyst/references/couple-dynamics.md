# 双人适配分析规则

## 1. 八个适配维度定义与评分依据

每个维度评分 1-10。评分基于双方认知功能分数和依恋类型的交叉分析。

### 1.1 共鸣（Resonance）
**定义**：两人在认知和情感层面自然共振的程度——哪些功能使他们"说同一种语言"。

**评分依据**：
- 双方高位功能（T1-T2）的重叠度：每重叠一个高位功能 +2 分
- 双方低位功能（T4）的重叠度（共享短板也是一种理解） +1 分
- 内倾/外倾态度的一致性 +1 分
- 判断-感知偏好的相似性 +1 分

### 1.2 平衡/互补（Balance）
**定义**：一方的强项补足另一方弱项的程度。

**评分依据**：
- 识别 4 对关键互补配对（一方 T1-T2 对应另一方 T4-T3）
- 每对显著互补（差距 >15） +2 分
- 每对中等互补（差距 10-15） +1 分

### 1.3 矛盾（Conflict）
**定义**：结构性冲突的数量、频率和强度。分数越低表示矛盾越多/越严重。

**评分依据**：
- Fe 差距 >20 → -3 分（情感语言不通——最严重的结构性矛盾）
- 依恋策略冲突（焦虑 vs 疏离）→ -2 分
- 直觉方向张力（Ni 收敛 vs Ne 发散，差距 >15）→ -1 分
- 判断功能冲突（Fi 价值观 vs Ti 逻辑分析，差距 >15）→ -1 分

### 1.4 价值观兼容
**定义**：核心价值观的一致性程度。

**评分依据**：
- 双方 Fi 均 >50 +4 分（共享对"真实性"的重视）
- 双方直觉功能均 >50 +2 分（共享对"深度"的追求）
- 双方 Se 均 <50 +2 分（共享对物质/感官的低优先级）
- 价值观摩擦点存在（如 Ti vs Fi 判断冲突） -1 分/个

### 1.5 成长潜力
**定义**：关系促进双方心理发展的潜力。

**评分依据**：
- A 的最弱功能（T4）恰好是 B 的较强功能（T1-T2）+3 分/对
- B 的最弱功能恰好是 A 的较强功能 +3 分/对
- 双方依恋策略恰好形成"可被对方触发也可被对方修复"的配对 +2 分
- 年龄或发展阶段差异提供额外的成长张力 +1 分（不标注具体年龄）

### 1.6 情感沟通
**定义**：两人在情感表达和接收上的匹配程度。

**评分依据**：
- 双方 Fe 的均值 → 直接决定关系中的情感语言流畅度
- Fe 均值 >45 +7 分基础 + 差距修正
- Fe 均值 35-45 +5 分基础
- Fe 均值 <35 +3 分基础
- Fe 差距 >15 → -2 分
- 一方 Fe <30 → -3 分（结构性情感失语）

### 1.7 日常生活
**定义**：共同维护生活秩序的兼容度。

**评分依据**：
- 双方 Si 均值 >50 +7 分基础
- 双方 Si 均值 40-50 +5 分基础
- 双方 Si 均值 <40 +3 分基础（两人都不擅长）
- 双方 Se 均值 <45 → -1 分（共同活动选项受限）
- 至少有一方 Te >55 +1 分（有执行者）

### 1.8 身体亲密
**定义**：身体和感官层面的兼容性和潜力。

**评分依据**：
- 双方 Se 均值 → 决定身体亲密的自然驱动力
- Se 均值 >50 +7 分基础
- Se 均值 40-50 +5 分基础
- Se 均值 <40 +3 分基础
- Fe 低 + Se 低的组合 → 身体亲密可能是绕过 Fe 的"备用情感通道" +1 分

## 2. 四阶段恋爱过程推演模板

### 2.1 阶段命名（不包含时间信息）
- 阶段一：智识蜜月（核心动力：高位功能的共鸣）
- 阶段二：差异浮现（核心动力：低位功能差异开始被感知）
- 阶段三：模式固化（核心动力：依恋策略和冲突模式的建立）
- 阶段四（分岔）：成长路径 vs 消耗路径

### 2.2 每个阶段必须包含
1. 双方体验的概述
2. 至少 1 个具体场景对话示例（📋 场景）
3. 功能动力学解释
4. 此阶段的"盲区"——在当下不被意识到但事后回看清晰的事情

### 2.3 阶段三的逻辑检验要求
必须回答：为什么当前的循环是"稳定"的？双方各自从循环中获得了什么隐性满足（二次获益）？打破循环需要什么条件？

## 3. 典型冲突分析模板

每个冲突必须包含：
- **标题**：功能碰撞标签 —— X(分数) vs Y(分数)
- **统计标签**：频次 / 破坏性 / 可解性（各分为高/中/低）
- **场景**：📋 格式，包含具体对话（A: / B: 格式，区分内心独白用斜体）
- **分析**：功能根源解释、双方各自在自己的框架内如何"正确"、误解如何产生

## 4. 典型风险分析模板

每个风险必须包含：
- **风险等级**：高/中/低
- **风险描述**：机制解释
- **场景升级示例**：初期→中期→后期的升级路径（如果适用）
- **前置信号**：伴侣可以观察到的预警信号

## 5. 解决办法模板

每个方法必须包含：
- **目标**
- **具体步骤**
- **对话示例**：A 和 B 的完整对话（成功执行该方法的情境）
- **适用条件**：什么情况下这个方法可能失败
