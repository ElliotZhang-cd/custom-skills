# 复杂系统分析

## 概述

运用五种集成理论（Systems/Info-Control/Game/Self-Org/Catastrophe）映射结构、追踪反馈动态、预测临界跃迁并识别高杠杆干预点。

核心原则：这是一个自指闭环，而非线性清单。元认知贯穿全程，每次迭代可能扩展或收缩系统边界。

## 何时使用

- 平台生态、市场动态、组织变革
- 含反馈回路与涌现行为的多主体问题
- 逼近临界点、相变或崩溃的系统
- 具有网络效应与非线性动力学的战略情境

**快速检验**：若以下 ≥3 项适用，则启用本框架：
- 多交互主体
- 反馈回路
- 涌现属性
- 策略博弈
- 潜在突变可能

**勿用场景**：
- 简单线性问题（<5 个交互元素，无反馈）
- 纯技术故障排查
- 纯数学问题
- 个人建议
- 用户明确要求其他方法

## 分析流程（认知闭环）

复制此清单并追踪进度。注意：步骤 4/5/6 均可跳过，若其理论失效条件不满足。

```
□ 界定：系统边界与层次（系统论）
□ 感知：信息量化与不确定性（信息论）
□ 调节：偏差纠正与实时控制（一阶控制）
□ 互动：多主体策略博弈（博弈论）— 若不满足条件则跳过
□ 涌现：自组织动力学（自组织理论）— 若不满足条件则跳过
□ 跃迁：临界风险与突变（突变论）— 若不满足条件则跳过
□ 反思：质疑目标/模型/规则（二阶控制/元认知）
□ 迭代：必要时回到"界定"，边界可能已改变
```

**步骤 1 — 界定（系统论）**：划定系统边界、识别子系统、分析网络拓扑、标注层级（微观→中观→宏观）、识别涌现属性。若边界不清晰，停止并询问用户。

**步骤 2 — 感知（信息论）**：识别观测内容、评估熵与信噪比、判断信息价值。若噪音超过信道容量（C < H），标记为不可精确量化，降低确定性声明。

**步骤 3 — 调节（一阶控制）**：定义目标态、当前态、误差信号、反馈类型与延迟。若延迟超过系统响应时间，标记为失控，转向博弈或自组织分析。

**步骤 4 — 互动（博弈论）**：识别参与者、目标、策略集、收益结构与均衡。仅在存在策略互依时应用。若无互依，降级为统计描述。

**步骤 5 — 涌现（自组织理论）**：检查三条件（开放性、远离平衡、非线性耦合）。若满足，识别序参量、时间线及当前阶段。若不满足，秩序不会自发涌现，跳过此层。

**步骤 6 — 跃迁（突变论）**：评估势函数、多稳态、控制参数及预警信号。若系统无势函数或维度 >4，改用统计物理方法描述临界现象。

**步骤 7 — 反思（元认知）**：质疑目标合理性、模型准确性、策略有效性。必要时升级规则或回到步骤 1。

## 认知层次

| 层次 | 功能 | 对应理论 |
|------|------|---------|
| 操作层 | 维持既定目标 | 信息论 + 一阶控制 |
| 结构层 | 演化新秩序 | 博弈论 + 自组织 + 突变论 |
| 认知层 | 升级系统本身 | 二阶控制（元认知） |

元认知贯穿全程，负责质疑和修正下层目标与规则。

## 理论边界与失效条件

| 理论 | 失效条件 | 应对策略 |
|------|---------|---------|
| 系统论 | 元素无相互作用（退化为集合论） | 降级为简单枚举 |
| 信息论 | 噪音 > 信道容量（C < H） | 标记不可观测，降低量化声明 |
| 控制论 | 延迟 > 响应时间 / 系统不可控 | 标记失控，转向博弈或自组织 |
| 博弈论 | 主体完全随机或完全最优 / 偏好不可传递 | 降级为统计描述 |
| 自组织 | 处于平衡态 / 线性耦合 / 封闭系统 | 跳过，秩序不会自发涌现 |
| 突变论 | 无势函数 / 控制变量 > 4维 / 强噪音淹没 | 改用统计物理临界相变理论 |

**框架整体边界**：
- 不能精确预测混沌系统长期行为
- 不能在多项式时间内求解 NP 完全问题
- 不能回答不可判定问题（如停机问题、自指悖论）

详细数学基础与跨学科验证见 [boundaries.md](boundaries.md)。

## 输出结构

始终按此结构输出。若某理论被跳过，显式注明：

```markdown
## [系统名称] 完整分析

### 执行摘要
[3-5 句：系统类型、当前阶段、关键发现、主要建议方向、风险等级。]
[若系统处于混沌区域或临界点附近，必须声明：以下预测为概率性而非确定性。]

### 1. 系统边界与结构
### 2. 信息与控制架构
### 3. 战略博弈分析
[若跳过："博弈论不适用：无策略互依关系。降级为统计描述。"]
### 4. 自组织动力学
[若跳过："自组织条件不满足：[具体条件缺失]。秩序不会自发涌现。"]
### 5. 临界性与突变风险
[若跳过："突变论不适用：[原因]。改用统计物理描述临界现象。"]
### 6. 综合建议
### 7. 局限性与不确定性
```

## 理论速查

| 理论 | 核心问题 | 关键产出 | 不适用时 |
|------|---------|---------|---------|
| 系统论 | 内外边界？ | 边界图、拓扑、涌现属性 | 元素无交互 |
| 信息论 | 如何感知？ | 熵、互信息、信道容量 | 噪音超限 |
| 控制论 | 如何调节？ | 反馈回路、延迟、元认知 | 不可控/延迟过大 |
| 博弈论 | 主体如何互动？ | 均衡、稳定性 | 无策略互依 |
| 自组织 | 秩序如何涌现？ | 序参量、时间线 | 平衡态/线性/封闭 |
| 突变论 | 何时跃迁？ | 临界点、预警信号 | 无势函数/维度>4 |

## 质量检查清单

提交前验证：

```
□ 已应用的每种理论均满足其前提条件（未应用的已明确说明原因）
□ ≥2 条反馈回路，含方向(+/-)与时间常数
□ ≥3 项可测指标，含当前值+预警阈值+检查频率
□ 所有机制已标注时间尺度
□ 干预已按杠杆排序（高/中/低），且附带置信度声明
□ 不确定性与假设已明确陈述
□ 尽量量化，附单位；无法量化的已声明原因
□ 若涉及混沌/临界区域，已声明预测的或然性
```

## 常见错误

Agent 在压力下常犯以下错误，必须避免：

| 错误 | 后果 | 纠正 |
|------|------|------|
| 跳过适用性检查，强行套用全部五论 | 在不适用场景输出伪科学分析 | 每个理论必须先过适用性检查 |
| 信息不足时仍给出精确数值预测 | 制造虚假确定性，误导决策 | 噪音 > 容量时，声明置信度为 low |
| 忽略元认知检查点，直接输出结论 | 边界/目标错误未被发现 | 步骤 3 和步骤 7 的检查点不可跳过 |
| 不声明预测或然性 | 用户误以为可精确预测混沌系统 | 涉及混沌/临界区域时，强制声明或然性 |
| 用"精神而非字面"绕过规则 | 规则形同虚设 | 违反字面 = 违反精神，必须重写 |

## 何时停止

- 系统为简单线性（<5 个交互元素，无反馈）
- 纯技术问题（如何配置 X）
- 用户明确要求其他方法
- 步骤 1 后边界仍不清晰 → 请用户澄清范围
- 信息噪音超过信道容量 → 标记为不可精确分析，停止强行量化
